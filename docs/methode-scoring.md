# Méthode de scoring

Ce document propose une méthode simple pour tester le moteur local de scoring avant toute automatisation.

Le score total est calculé sur 100 points.

## Répartition proposée

### 1. Correspondance des compétences - 30 points

Evaluer si les compétences demandées dans l'offre sont présentes dans le profil.

Exemples :

- HTML, CSS, JavaScript.
- SQL ou logique de données.
- communication écrite.
- rigueur.
- travail en équipe.
- support utilisateur ou relation client.

Score indicatif :

- 25 à 30 : forte correspondance.
- 15 à 24 : correspondance partielle.
- 0 à 14 : correspondance faible.

### 2. Correspondance des outils - 20 points

Evaluer les outils explicitement cités dans l'offre.

Exemples :

- outils web ;
- Git ;
- SQL ;
- outils de suivi ;
- outils no-code ;
- Google Sheets ou formulaires.

Score indicatif :

- 16 à 20 : outils majoritairement connus ou proches du profil.
- 8 à 15 : plusieurs outils proches, mais certains manquent.
- 0 à 7 : outils peu alignés.

### 3. Type de contrat et localisation - 20 points

Evaluer la compatibilité pratique.

Critères :

- stage accepté ;
- convention possible ;
- localisation compatible ;
- télétravail éventuel ;
- temps de travail compatible ;
- date de début compatible.

Score indicatif :

- 16 à 20 : très compatible.
- 8 à 15 : compatible avec points à vérifier.
- 0 à 7 : contraintes importantes.

### 4. Niveau junior ou stage accepté - 15 points

Evaluer si l'offre accepte un profil en formation ou en reconversion.

Score indicatif :

- 13 à 15 : junior/stage explicitement accepté.
- 7 à 12 : junior possible mais pas totalement clair.
- 0 à 6 : expérience confirmée demandée.

### 5. Cohérence avec les objectifs professionnels - 15 points

Evaluer si l'offre aide Hassan à avancer vers son objectif.

Critères :

- développement web ;
- automatisation ;
- outils numériques ;
- automatisation ou no-code ;
- projets concrets ;
- montée en compétences.

Score indicatif :

- 13 à 15 : très cohérent.
- 7 à 12 : intéressant mais périphérique.
- 0 à 6 : peu lié aux objectifs.

## Seuils de décision

- 85 à 100 : très bonne opportunité.
- 70 à 84 : opportunité intéressante.
- 50 à 69 : à examiner.
- Moins de 50 : peu prioritaire.

## Règles de prudence

- Le score ne doit pas inventer de compétences absentes du profil.
- Les compétences souhaitées doivent peser moins lourd que les compétences requises.
- Une offre intéressante mais trop administrative doit être signalée clairement.
- Une validation humaine reste obligatoire avant toute candidature.
- Le score doit toujours être accompagné d'une explication courte.
